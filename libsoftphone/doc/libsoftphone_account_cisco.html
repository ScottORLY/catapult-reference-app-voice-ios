<ciscoRedundantRouteSets><RouteSet>…</RouteSet><RouteSet>…</RouteSet>…</ciscoRedundantRouteSets>

No default value.

Enables Cisco redundancy functionality.

Note that a single RouteSet node describes route to a single SIP server. So if there are N redundant SIP servers (excluding the primary SIP server) then there should be N RouteSet nodes.

The format of the RouteSet node is as follows:

<RouteSet>
  <Route host="hostname[:port]" transport="…" />
  …
</RouteSet>

where the possible values for the transport attribute are udp, tcp, tls and tls+sip:.
